What is Base64 Encoding?

Base64 is a binary-to-text encoding scheme that represents binary data using only 64 printable ASCII characters (A–Z, a–z, 0–9, +, /). It was designed to safely transmit binary data over channels that only support text, such as email (MIME) or JSON APIs.

Every 3 bytes of input are converted into 4 Base64 characters, resulting in approximately a 33% increase in size. An = character is appended as padding when the input length is not a multiple of 3.

When Should You Use Base64?

  • Embedding images in HTML/CSS: src="data:image/png;base64,..." avoids a network request.
  • API authentication: HTTP Basic Auth encodes user:password in Base64 in the Authorization header.
  • JSON payloads: Binary files (PDFs, images) can be embedded in JSON by encoding them as Base64 strings.
  • JWT tokens: The header and payload sections of a JWT are Base64url-encoded (a URL-safe variant).
  • Email attachments (MIME): The MIME standard uses Base64 to encode binary attachments in plain-text email bodies.

Base64 vs Base64url

Standard Base64 uses + and /, which are special characters in URLs. Base64url replaces them with - and _ to make the output safe for use in URLs and filenames without percent-encoding. JWTs use Base64url. This tool uses standard Base64 (btoa / atob).
