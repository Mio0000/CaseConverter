What is a Unix Timestamp?

A Unix timestamp (also called epoch time) is the number of seconds that have elapsed since 1970-01-01 00:00:00 UTC, known as the Unix epoch. It is a timezone-independent, language-agnostic way to represent a point in time as a single integer.

The current Unix timestamp is always increasing by 1 every second. As of 2024, it is approximately 1,700,000,000. Many systems also use millisecond timestamps (multiply by 1000), which you can recognize because they are around 13 digits rather than 10.

Why Do Developers Use Unix Timestamps?

  • Timezone-safe: Timestamps are always UTC-based, eliminating ambiguity across timezones.
  • Easy arithmetic: Adding 86400 to a timestamp gives exactly one day later — no calendar logic needed.
  • Database storage: Storing an integer is more compact and faster to index than a datetime string.
  • API contracts: REST and GraphQL APIs commonly exchange timestamps as Unix integers (e.g. JWT exp, Stripe events).
  • Log files: System logs (syslog, nginx) often record Unix timestamps for precision and sortability.

Seconds vs Milliseconds

Unix time is traditionally in seconds (10 digits as of 2024). JavaScript's Date.now() returns milliseconds (13 digits). When you receive a timestamp from an API, check its magnitude: if it's around 1.7 × 10¹² it's in milliseconds; if it's around 1.7 × 10⁹ it's in seconds. This tool auto-detects both.
