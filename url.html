What is URL Encoding?

URL encoding (also called percent-encoding) replaces unsafe ASCII characters with a % sign followed by two hexadecimal digits. For example, a space becomes %20 and & becomes %26. This ensures that special characters in URLs do not conflict with the URL syntax itself.

encodeURIComponent vs encodeURI vs Full URL Encode

  • encodeURIComponent — Encodes everything except letters, digits, and - _ . ! ~ * ' ( ). Use this for query parameter values: q=hello%20world.
  • decodeURIComponent — Reverses encodeURIComponent. Use when reading a URL-encoded query string.
  • Full URL encode (encodeURI) — Preserves the URL structure (://, /, ?, =, &) while encoding only characters that are completely illegal in URLs. Use this for a complete URL.

Common Use Cases

  • Query strings: Spaces and special chars in search terms must be encoded: q=C%2B%2B+tutorial.
  • API calls: Path segments or parameter values containing slashes, ampersands, or equals signs require encoding.
  • Redirect URLs: A redirect_uri parameter must itself be URL-encoded so it isn't parsed as part of the outer URL.
  • Form data: HTML form submissions use a similar scheme called application/x-www-form-urlencoded.
